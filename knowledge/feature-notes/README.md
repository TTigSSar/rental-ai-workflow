# Feature Notes

One note per NON-trivial feature, written at the close-feature step. Trivial changes (styling, small fixes) get no note — their trace is the commit.

File name: `YYYY-MM-<slug>.md`. Required header links the note to the rest of the knowledge graph:

```
# <Feature Name>
Status: Completed | Decisions: ADR-XXX[, ...] | Mistakes: M-XXX[, ...] | PR: #NN | Issues: #NN

Goal → initial plan → alternatives considered → what went wrong → final solution → lessons.
```
